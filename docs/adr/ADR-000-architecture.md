# Контекст

Система реализует AI-функциональность с использованием:

* LLM
* RAG
* Tools
* внешних интеграций

Требуется:

* поддержка sync и async execution
* независимость от конкретных провайдеров
* управляемость cost / latency / quality
* расширяемость

# Решение

Используется единая архитектура со следующими принципами.

## Single Entry Point

Все запросы поступают через единый API.

Режим выполнения определяется параметрами запроса:

* sync → немедленный ответ
* async → постановка задачи в очередь

Бизнес-логика не зависит от способа выполнения.

## Decision First

Перед выполнением запроса он всегда обрабатывается через входную ноду графа определяется стратегия обработки:

* LLM
* RAG
* Tools

и/или дальнейшее поведение процесса.

Решение принимается явно через Decision Policy.

## Hexagonal Architecture

Взаимодействие между бизнес-логикой и инфраструктурой осуществляется через Ports - Rounting Adapter - Adapter.

Сервисы не зависят от:

* LLM провайдеров
* Vector DB
* MCP серверов
* внешних API

## Retrieval First

Приоритет обработки:

1. Tools
2. RAG
3. LLM

LLM используется только когда задача не может быть решена более детерминированным способом.

# Последствия

## Плюсы

* независимость от поставщиков
* простая замена инфраструктуры
* высокая тестируемость
* контроль стоимости

## Минусы

* большее количество слоев
* усложнение архитектуры

# Альтернативы

* прямые вызовы LLM
* LangChain-only pipelines
* LLM-first architecture
