# Контекст

Prompt является частью AI-системы и должен иметь единообразную структуру.

Отсутствие стандарта приводит к:

* неоднородности prompt-файлов
* усложнению сопровождения
* сложностям ревью
* проблемам повторного использования

# Решение

Все prompt-файлы должны использовать единую структуру.

## Структура промпта

`# Metadata (YAML Front Matter)`
---
prompt_id: support_assistant
version: 1.0.0
author: <author>
created_at: YYYY-MM-DD
updated_at: YYYY-MM-DD
status: draft
description: >
  Краткое описание назначения prompt.
tags:
  - support
  - customer
owner: <team>
---

`# Role`
Описание роли модели.

`# Goal`
Какой результат должен быть получен.

`# Context`
Бизнес-контекст задачи.

`# Inputs`
Описание входных данных.

`# Instructions`
Пошаговые инструкции модели.

`# Constraints`
Ограничения и запреты.

`# Output Format`
Описание ожидаемого результата.

`# Failure Handling`
Описание поведения при невозможности корректного выполнения задачи.

`# Evaluation Criteria`
Критерии качества результата.

# Последствия

## Плюсы
* единый формат prompt-файлов
* упрощение ревью
* повышение переиспользуемости
* упрощение сопровождения

## Минусы
* дополнительная структура для небольших prompt-файлов

## Альтернативы
* свободная структура prompt-файлов (отклонено)
* отсутствие стандарта (отклонено)
