# Контекст
Запрос пользователя может быть обработан разными способами:
- LLM
- RAG
- Tools

Нужен управляемый выбор стратегии.

# Решение
Введён отдельный слой:

decision_policy

Алгоритм:
1. deterministic rules (actions → tools)
2. heuristics (knowledge → RAG)
3. fallback → LLM

LLM не используется для принятия решения по умолчанию.

# Последствия

## Плюсы
- предсказуемость поведения
- explainability
- контроль затрат

## Минусы
- необходимость поддержки правил
- возможная сложность при росте сценариев

## Альтернативы
- LLM routing (отклонено — нестабильно и дорого)
- статический pipeline (отклонено — негибко)
