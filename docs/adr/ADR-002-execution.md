# Контекст
Требуется поддержка:
- синхронной обработки (HTTP response)
- асинхронной обработки (очереди, event-driven)

# Решение
Введён abstraction:

executor

Режим определяется на уровне запроса:
- sync → immediate execution
- async → enqueue job + task_id

Бизнес-логика не зависит от режима исполнения.

# Последствия

## Плюсы
- единый код для sync/async
- гибкость масштабирования
- упрощение API

## Минусы
- требуется инфраструктура очередей
- сложнее дебаг

## Альтернативы
- отдельные сервисы (отклонено — дублирование логики)
