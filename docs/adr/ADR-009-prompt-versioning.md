# Контекст
Prompt = часть бизнес-логики

Проблемы:
- изменения не отслеживаются
- сложно воспроизводить поведение
- невозможно дебажить

# Решение

Вводится versioned prompt registry:

## Структура
- prompt_id
- version
- template
- metadata (описание, автор, дата)

Пример:

prompt_id: "support_assistant"
version: "v2"

## Использование
- всегда указывать:
  - prompt_id
  - version

## Хранение
- в коде (platform/prompts)
- либо в config storage (опционально)

## Правила

- Любое изменение prompt:
  → новая версия (immutable)
- Запрещено:
  - менять существующую версию
- Логирование:
  - version обязателен в trace

## Дополнительно

- A/B тестирование через feature flags
- rollback через переключение версии

# Последствия

## Плюсы
- воспроизводимость
- контроль изменений
- безопасные эксперименты

## Минусы
- дополнительная дисциплина
- управление версиями

## Альтернативы
- хранение inline (отклонено)
- без версий (отклонено)
