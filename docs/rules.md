# АРХИТЕКТУРА

- Строго следуй docs/structure.md
- services слой НЕ зависит от adapters
- все внешние вызовы (LLM / DB / API) → только через ports
- запрещены прямые вызовы adapters из бизнес-логики
- execution mode (sync/async) → только через ExecutorPort

Если архитектура мешает:
→ сначала предложи изменение architecture.md

# БАЗОВЫЕ ПРАВИЛА

- Нет хардкода → константы
- Понятные имена
- DRY
- SOLID
- Разделяй:
  - бизнес-логику
  - инфраструктуру
  - I/O

Комментарии:
- объясняй ПОЧЕМУ, а не ЧТО

Все внешние операции:
- идемпотентны
- timeout обязателен
- retry: bounded + backoff + jitter

Ошибки:
- retryable / non-retryable / fatal

Feature toggle обязателен для критичных изменений

# БАЗА ДАННЫХ / ORM (если используется)

## Миграции
- только через Liquibase
- обязательно: rollback, блокировки, время, backfill
- большие таблицы: expand → migrate → contract

## Запросы
- `SELECT *` запрещён
- SQL параметризованный
- индексы обязательны
- проверка через EXPLAIN ANALYZE

## Операции
- массовые → батчи + WHERE
- избегать full scan

## Транзакции
- короткие
- без внешних вызовов
- явные границы

## JPA
- LAZY по умолчанию
- избегать N+1
- нет findAll без LIMIT
- нет логики в entity

## Надёжность
- идемпотентность
- upsert / unique constraints

# KAFKA (если используется)

- контракт обязателен
- consumer идемпотентен
- commit после успеха
- retry + DLQ обязательны
- обработка duplicates / out-of-order
- модель: at-least-once

---

# КЭШ (если используется)

- явная политика: TTL, invalidation
- не источник истины
- защита от stampede
- fallback обязателен

# LLM / AI (если используется)

Следовать AI Runtime Contract.

Ключевые правила:

- LLM только если нельзя детерминированно
- retrieval-first
- минимизировать количество вызовов

Каждый LLM-вызов:
- structured output обязателен
- timeout + retry
- token budget

Tools:
- только через ToolPort
- строгая схема
- allowlist

RAG:
- обязательный provenance
- фильтрация источников

Безопасность:
- защита от prompt injection
- защита от tool injection

# БЕЗОПАСНОСТЬ

- принцип наименьших привилегий

- allowlist форматов
- маскирование чувствительных данных

SQL:

- audit trail
- логирование

Запрещено:
- выполнение произвольного кода
- небезопасная десериализация
- утечка секретов
