# RULES:
  Это набор правил и ограничений для AI-асситента.

# SKILLS:
  Это ПРОСТО перечень инсрукции "Сделай то-то и то-то". По сути это как блок Instructions в промпте, просто вынесенный в отдельный файл, который может вызываться как вручную, так и автоматически AI-ассистентом.
  Если проводить аналогию - это как вынести общее поведение нескольких методов в отдельный метод.

  Пример:
  - Нужно создать порядок проведения code-review. Вместо того, чтобы перегружать rules, в котором есть требование провести code-review, это выносится в SKILL.
  - Агент (ну или субагент) сам подхватит skill и выполнит порядок действий.
  - При необходимости можно вызвать skill вручную

# Subagents (кроме Windsurf)
  Если проводить аналогию - это как вызов метода в отдельном потоке из другого метода. Родительский агент, запущенный AI-ассситентом, может вызывать субагентов.
  
  Главные фишки субагентов:
  - Свой изолированный контекст, который не переполняет контекст родительского агента.
  - Свой изолированный набор инструментов и правил.
  - Возможность указать, какую модель должен использовать субагент.
  - Фоновый режим работы (например, можно запустить субагент для сканирования файлов и получить результат в фоне, не блокируя основной агент).
  
  Т.е. субагент помогает решить такие проблемы как:
  - Переполнение контекста родительского агента (а значит увеличение качества его работы)
  - Расход дорогих токенов (а значит бюджетов). Например, основной агент использует дорогую модель, а для субагента, который просто должен найти какую то информацию (просканировать файлы и найти нужный) можно указать дешевую модель.
  - Низкая скорость выполнения задачи (например, можно запустить субагент для сканирования файлов и получить результат в фоне, не блокируя основной агент).

  - Базовые (бэст-практис):
    - code-reviewer
    - test-generator
    - documentation-generator
    - security-scanner

# Пререквизиты

- Установи менджер uv
  Windows: 
  ```
  powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
  ```

  Linux:    

- Установи Python:
  ```
  uv python install
  ```

- Создай venv
  ```
  uv venv
  ```

- Синхронизируй зависимости
  ```
  uv sync
  ```

- Установи node.js для использования менеджера пакетов npm
  - Windows / Linux: скачай оф установщик с `https://nodejs.org/en/download`
  - Получи API-key для Context7: `https://context7.com/dashboard`
  - Добавь системную переменную `CONTEXT7_API_KEY` и запиши в нее полученный API-key
  - Установи MCP-сервер для Context7 
    ```
    npx ctx7 setup
    ```
  - Обрати внимание! Установщик запросит указать инструмент, в котором планируется использовать MCP. Автоматически усьтановщик создаст SKILL, RULE и mcp.json в каталоге users/.../

- Для Devin/Windsurf project-level пока не поддерживается, поэтому надо к себе в папку копирнуть: 
  ```
  copy .devin/mcp_config.json %USERPROFILE%\.codeium\windsurf\mcp_config.json
  ```
