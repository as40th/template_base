# Context7 — обязательный источник документации

Всегда используй Context7 как основной источник актуальной документации при работе с кодом, библиотеками, фреймворками, SDK, API и инструментами разработки.

Не полагайся исключительно на свои обучающие данные, если задача связана с использованием сторонних библиотек, фреймворков или внешних API. Перед генерацией кода, рефакторингом, исправлением ошибок или архитектурными рекомендациями сначала получай актуальную документацию через Context7.

## Когда использовать Context7

Используй Context7 автоматически в следующих случаях:

* Генерация нового кода с использованием библиотек или фреймворков.
* Анализ существующего кода, использующего внешние зависимости.
* Исправление ошибок и отладка.
* Рефакторинг.
* Настройка проектов и конфигурации.
* Работа с SDK и API.
* Создание примеров использования библиотек.
* Проверка актуального синтаксиса и сигнатур методов.
* Проверка изменений между версиями библиотек.
* Любая задача, где используются зависимости из `package.json`, `requirements.txt`, `pyproject.toml`, `Cargo.toml`, `pom.xml`, `build.gradle` и других менеджеров пакетов.

Даже если пользователь напрямую не упоминает библиотеку, но она обнаружена в проекте, используй Context7 для получения актуальной информации.

## Исключения

Context7 можно не использовать только если:

* задача не связана с программированием;
* используются исключительно встроенные возможности языка без сторонних библиотек;
* требуется работа исключительно с пользовательским кодом без обращения к внешним зависимостям.

## Порядок работы

### Шаг 1. Определение библиотеки

Определи все библиотеки и фреймворки, которые участвуют в задаче.

Источники определения:

* запрос пользователя;
* код проекта;
* файлы зависимостей;
* импорты в открытых файлах.

### Шаг 2. Поиск идентификатора библиотеки

Вызови `resolve-library-id`:

* `libraryName` — название библиотеки;
* `query` — полный контекст текущей задачи.

### Шаг 3. Выбор документации

При выборе результата отдавай приоритет:

1. официальным пакетам;
2. официальной документации;
3. документации для конкретной версии проекта;
4. результатам с более высоким рейтингом качества.

Если версия указана в проекте или запросе, всегда используй документацию именно этой версии.

### Шаг 4. Получение документации

Вызови `query-docs`:

* `libraryId` — выбранный идентификатор библиотеки;
* `query` — текущая задача или вопрос.

### Шаг 5. Использование документации

На основе полученной документации:

* генерируй код;
* исправляй ошибки;
* предлагай архитектурные решения;
* создавай примеры использования;
* объясняй работу API.

При наличии различий между обучающими данными и документацией Context7 всегда отдавай приоритет данным Context7.

## Приоритет

Context7 является основным источником истины для всех внешних библиотек и фреймворков.

Если существует возможность получить актуальную информацию через Context7, используй её до генерации ответа или кода.
