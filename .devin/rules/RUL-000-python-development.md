# Управление зависимостями Python

Для всех новых Python-проектов используй современный стандарт управления зависимостями на основе `pyproject.toml`.

## Обязательные правила

- Не создавай `requirements.txt` для новых проектов без явного запроса пользователя.
- Не используй `pip install -r requirements.txt` в примерах, если пользователь не работает с существующим проектом, использующим этот подход.
- Используй `pyproject.toml` как единственный источник описания зависимостей проекта.
- Предпочитай использование `uv` для управления зависимостями и виртуальным окружением.
- Для новых проектов генерируй структуру, совместимую с `uv`.

## Добавление зависимостей

Для установки новых зависимостей используй:

```bash
uv add <package>
```

# Типизация

## Обязательно

* строгая типизация во всех слоях
* использование ABC для контрактов
* использование TypedDict / dataclass / Pydantic при необходимости

---

## Запрещено

```python
Any
```

---

## Если тип неизвестен

Использовать:

* object
* ABC
* TypeVar
* Generic

---

# Async-first

## Запрещено в async-коде

```python
requests
time.sleep
subprocess.run
```

## Использовать

```python
httpx.AsyncClient
asyncio.sleep
asyncio.create_subprocess_exec
```

---

## Требование

* никакого blocking I/O в async flow

# Логирование

Сконфигурируй логирование в приложении с использованием формата:
```
%(asctime)s - %(name)s - %(levelname)s - %(filename)s:%(lineno)d - %(message)s
```

---

# Обработка ошибок

## Запрещено

```python
except:
    pass
```

---

## Требуется

Явная обработка исключений:

* retryable
* non-retryable
* fatal

---

# Архитектурные принципы

## Предпочитать

* маленькие функции
* композицию вместо наследования
* dependency injection
* явные зависимости

## Избегать

* global state
* скрытых side-effects
* magic values

---

# Definition of Done

Код считается валидным если:

* нет Any
* нет blocking I/O в async
* корректная типизация
* ошибки обработаны явно
* зависимости обоснованы
