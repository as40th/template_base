# Определение библиотеки

Определи, какие внешние библиотеки или фреймворки используются в задаче.

Источники определения:

- вопрос пользователя;
- существующий код;
- зависимости проекта (`package.json`, `pyproject.toml`, `requirements.txt` и т.д.);
- импорты в коде.

Если внешние зависимости обнаружены — Context7 обязателен.

---

# Определение ID библиотеки

Вызови `resolve-library-id` со следующими параметрами:

- `libraryName`: название библиотеки, извлечённое из контекста задачи
- `query`: полный текст запроса пользователя или описание задачи

---

# Выбор правильной документации

Из результатов `resolve-library-id` выбери наиболее подходящий вариант:

- приоритет точному совпадению названия библиотеки;
- затем — официальной документации;
- затем — наиболее высокому benchmark score;
- если указана версия (например, React 19, Next.js 15), обязательно выбирай документацию, соответствующую версии.

Если есть сомнения между несколькими вариантами — выбирай наиболее современную и официально поддерживаемую версию.

---

# Получение документации

Вызови `query-docs` со следующими параметрами:

- `libraryId`: выбранный ID библиотеки (например, `/vercel/next.js`)
- `query`: конкретная задача пользователя

---

# Использование документации

Используй полученную документацию как основной источник истины при генерации ответа.

На основе документации:

- генерируй код;
- объясняй API;
- предлагай архитектурные решения;
- исправляй ошибки;
- обновляй устаревшие подходы.
