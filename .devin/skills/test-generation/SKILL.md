# Goal

Сгенерировать набор тестов для указанного кода или модуля.

---

# Process

## 1. Analyze target

Определи:

- публичные функции / классы
- входы и выходы
- бизнес-инварианты
- критические сценарии
- если пользователь не указал тип тестов - ОБЯЗАТЕЛЬНО остановись и уточни, какие именно тесты ему нужны (unit, integration, e2e)
- если пользователь не указал техстэк - ОБЯЗАТЕЛЬНО остановись и уточни, какой техстэк следует (wiremock, testcontainers-python, pytest, etc.)

---

## 2. Identify test cases

Сгруппируй сценарии:

- happy path
- edge cases
- failure cases
- async / concurrency cases (если применимо)

---

## 3. Design test structure

Раздели тесты на:

- unit
- integration
- e2e (если требуется)

---

## 4. Define test behavior

Для каждого теста определи:

- входные данные
- ожидаемый результат
- поведение при ошибке

---

## 5. Generate test code

Сформируй тесты в соответствии с существующей структурой проекта.

---

## 6. Ensure coverage completeness

Проверь:

- покрыты ли ключевые ветки
- есть ли негативные сценарии
- есть ли async scenarios (если применимо)

---

# Output

- готовый тестовый код
- структура файлов
- без объяснений внутри тестов
